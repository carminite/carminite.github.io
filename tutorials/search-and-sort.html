Search and Sort

Just to get that E in Organizational Skills

One important aspect of programming is the ability to sift through data and organize it, which is achieved through searching and sorting. However, the problem is how to search for an item, and how to sort as efficiently as possible. This is where algorithms come into play.

Algorithms is a set of instructions that a computer follows to process data in a way to achieve a certain goal. This tutorial will focus on different algorithms for searching and sorting.

However, before we start, we need to define the measurements we use to determine the efficiency of an algorithm:

All algorithms scale with the number of items to search or sort. We denote this number with n.

Time and space are both written as O(x) (Big O notation), where x is some form of function.

  • Time Complexity: The amount of time an algorithm takes relative to other algorithms. For this tutorial, we will look at the average time complexity.
  • Worst Space Complexity: The most amount of temporary memory an algorithm will take up relative to other algorithms.
  • Stability (applies only to sorting): If two of the same values were input in a certain sequence, will they be output in that order?

Sorting

Almost all data can be sorted. Numbers can be sorted from least to greatest or greatest to least, Strings are sorted lexicographically (dictionary-style), and all objects have some type of attribute which can be quantified and thus sorted. However, the following sorting algorithms will be implemented with an int data type for convenience.

Bubble Sort

The simplest sorting algorithm.

Every adjacent pair of elements is compared. If the left element is greater than the right element, the two elements are swapped. Else, nothing is done. The algorithm then moves one index to look at the second pair. The resulting movements are like bubbles floating up and sinking down, hence the name.


When the algorithm cycles through the elements one time, it goes back to the beginning and cycles through again. The algorithm will only stop when it cycles through the whole set of data without making a swap.

Code:

void bubbleSort(int arr[])
{     int n = arr.length;
    for (int i = 0; i < n-1; i++)
        for (int j = 0; j < n-i-1; j++)
            if (arr[j] > arr[j+1])
            {
                // swap arr[j+1] and arr[j]
                int temp = arr[j];
                arr[j] = arr[j+1];
                arr[j+1] = temp;
            }
}

  • Average Time Complexity: O(n^2)
  • Worst Space Complexity: O(1)
  • Stability: Stable

Insertion Sort

Insertion sort executes as following:

  1. Iterate from index 1 onwards.
  2. Compare the current item to the item before it.
  3. If the item is less, then move the item to the left of the array until it is greater than the item before it.

Code:

void insertionSort(int arr[])
{
    int n = arr.length;
    for (int i = 1; i < n; ++i) {
        int key = arr[i];
        int j = i - 1;

        /* Move elements of arr[0..i-1], that are
        greater than key, to one position ahead
        of their current position */
        while (j >= 0 && arr[j] > key) {
            arr[j + 1] = arr[j];
            j = j - 1;
        }
        arr[j + 1] = key;
    }
}

  • Average Time Complexity: O(n^2)
  • Worst Space Complexity: O(1)
  • Stability: Stable

Searching

Search algorithms return the index of the item you search for. Because usually no temporary space is used, the only measurement we need is time complexity.

Linear Search

This algorithm simply iterates over the entire array to find an item.

Code:

int linearSearch(int[] arr, int n) {
    for (int i = 0; i < arr.length; ++i)
        if (arr[i] == n) return i;
    return 0;
}

Time Complexity: O(n)

Binary Search

Precondition: the array has to be sorted.

This search starts at the middle of the array and compares the search item with the value at the middle index. If equal, that index is returned. If greater or less, one half of the array (if search item is greater, then all indexes greater than the middle index, and vice versa for lesser) is removed and the search is then applied to the remaining values.

Code:

int binarySearch(int arr[], int l, int r, int x)
{
    if (r >= l) {
        int mid = l + (r - l) / 2;
        // If the element is present at the
        // middle itself
        if (arr[mid] == x)
            return mid;
        // If element is smaller than mid, then
        // it can only be present in left subarray
        if (arr[mid] > x)             return binarySearch(arr, l, mid - 1, x);
        // Else the element can only be present
        // in right subarray
        return binarySearch(arr, mid + 1, r, x);
    }
    // We reach here when element is not present
    // in array
    return -1;
}

Time Complexity: O(log(n))
