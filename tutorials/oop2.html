Advanced OOP

Inheritance and Polymorphism

Another property of object-oriented programming is that classes often are similar to other classes, and have similar attributes and behaviours. So instead of rewriting all the content, we use inheritance.

Implementation

public class Child extends Parent {

}

When a class extends another class, it inherits all the attributes and behaviours (field variables and methods) of that class, making it a subclass of its parent class (or superclass). This means that, unless there is override (which will be explained later), everything you could do in the superclass, you can do in the subclass.

For example, let’s say you have a superclass called Person.

public class Person {
    private String name;
    public Person(String n) {
        name = n;
    }
    public String getName() {
        return name;
    }
    public void setName(String n) {
        name = n;
    }
    public void greet() {
        System.out.println(“Hi”);
    }
}

You may recognize this from the Basic OOP tutorial.

But what if we don’t want a generic person, but a middle-aged Italian plumber?

public class Plumber extends Person {
    private int lives;
    public Plumber(String n) {
         super(n);
    }
    public void oneUp() {
        lives++;
    }
    @Override
    public void greet() {
        System.out.println(“It’s-a me, ”+this.getName());
    }
}

Let’s go over the new things we see.

1. super()

A subclass inherits from its superclass everything but its constructor, so super is used to solve this problem. In the program above, the constructor in Plumber will take the name of the Plumber as a parameter and pass it down to the Person constructor through super(), so the name variable gets assigned the name parameter. You can also use super to refer to attributes and behaviours in the superclass.

2. @Override Annotation

Sometimes, being specific as a subclass calls for change of the methods that it inherits. If the name and parameters of the method are the same in both classes, the subclass version is always executed, or overrides. The @Override annotation is used to organize and show more clarity.

3. this

The keyword this is used to refer to the object itself. This is especially useful when you are not able to directly access the attributes of a superclass (due to them being private) and thus need to use get() and set() methods instead (in this case, instead of using name which is private under Person, we use this.getName(). We can also use super.getName()).

So if we create a Plumber object and use a few methods:
Plumber mario = new Plumber(“Mario”);
System.out.println(mario.getName());
mario.greet();

The following will output:

Mario
It’s-a me, Mario

Interfaces

Sometimes, there are multiple attributes and behaviours required in a class, but using inheritance isn't convenient nor useful. Consider the following code segment.

public class Animal {
    int age;     public Animal(int a) {         age = a;     }     public void makeSound() {
        System.out.println("Insert generic animal sound here.");
    }
    public void run(int d) {
        System.out.println("Animal runs a distance of "+d+" meters.");
    }
}

Now let's make a Pig class.

public class Pig extends Animal {     public Pig(int a) {         super(a);     } }

There's a problem now. Pig can now access the methods makeSound() and run(), but Pigs have a unique animal sound:

If you were to just leave the Pig class and call the makeSound() method:

Pig wilbur = new Pig();
wilbur.makeSound();

A two-year-old would be able to see that this program has an issue.

That is one of the problems with inheritance. If a superclass method is abstract enough (we know it exists, but it varies greatly from one subclass to another), every single subclass would have to override it in its own class, which defeats the purpose of inheritance. Thus, there is another solution: polymorphism.

public interface Animal {
    public void makeSound();
    public void run(int d);
}

This is an interface. An interface is like an abstract class, where you are able to declare field variables and methods, but you cannot create an object from it. As well, notice that we don't implement the methods.

Let's make another Pig class.

public class Pig implements Animal {
    private int age;
    public Pig(int a) {
        age = a;
    }
    public void makeSound() {
        System.out.println("oink");
    }
    public void run(int d) {
        System.out.println("Pig ran "+d+"meters.");
    }
}

Just like how subclasses extend superclasses, classes can also implement interfaces. The classes themselves define what the interface methods do.

Advantages of Interfaces

Java doesn't support multiple inheritance; a subclass can only have one superclass. However, it can implement multiple interfaces, which makes them very versatile.

There is also a security component, where interfaces don't show the details of the methods it contains.
