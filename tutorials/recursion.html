Recursion

Programming's Matryoshka dolls

Recursion in programming is the concept of implementing a method that calls on itself, directly or indirectly. This can be connected straight to recursive functions in mathematics, like the Fibonacci sequence.

Structure

Typical recursion in the method usually has two or more parts, split into cases:

  1. The Base case
  2. The Recursive case(s)
To demonstrate this, let’s use the Fibonacci sequence. Assume that the method below will return the nth Fibonacci term:

public static int fibonacci(int n) {

The base case is typically a value in which all recursions should end on. In the Fibonacci sequence, we are always given that the first and second terms are 1. These would be the base cases.

if (n == 2 || n == 1) {
    return 1;
}

The recursive case (or cases) is the case where the method is called when calculating the return value. The Fibonacci sequence is built from the function:


In code, the implementation is:

return fibonacci(n-1)+fibonacci(n-2);

Notice the structural similarity between the code version and the math version.

Putting it all together we get:

public static int fibonacci(int n) {
    if (n == 2 || n == 1) { //Base case
        return 1;
    }
    return fibonacci(n-1)+fibonacci(n-2); //Recursive case
}

Benefits and Consequences

Compared to iterative functions (loops), recursion often requires less implementation. As well, many functions can only be achieved through recursion (like the Fibonacci and hailstone sequences).

However, there are several drawbacks. Because recursion calls on the same method multiple times, it often takes up a lot of memory and time. Consider that for every method call, variables have to be declared and assigned, calculations have to be made, and often more methods are called. All this takes up memory.
