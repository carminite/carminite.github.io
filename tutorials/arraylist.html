ArrayList

Arrays and Lists combined together

One of the problems with arrays is that they have a set length. No matter what you do with the array, you cannot add an index or take one away. This sometimes causes problems.

For example, say you had a todo list that you wanted to convert to an array.

You could technically use a String array to copy each element on the list into, leaving you with an array of length 2. However, what if you finished procrastinating? You would have to erase the first index from the array, but what to replace it with? null sounds like a good choice, but now what counts as the first thing on your list? The null or the second item, “Do everything else”?

Even worse, todo lists typically grow in length with the passage of time. What if you wanted to add another item to your todo list and your array was already full? Where do you put it? You could make another array with a greater length and copy the original array contents, but is that feasible?

This is where ArrayLists come in.

Implementation

First of all, let’s create an ArrayList object for our todo list.

ArrayList<String> todo = new ArrayList();

When you create an ArrayList, you have to specify what type of object you are going to fill it with, similar to arrays. This goes into the <> brackets. Notice how we didn’t need to say what the length of the ArrayList is.

Now let’s add an item.

todo.add(“Walk the dog”);

Our item appeared on the list, at index 0 (everything in programming is zero-indexed). Great. But you just remembered that you have to procrastinate before you walk the dog, so you add that:

todo.add(0, “Procrastinate”);

What’s different about these two add() methods is that the former doesn’t specify the index, while the latter does (note the 0 as a parameter). By default, add() inserts the item at the end of a list, but putting an index makes the ArrayList insert the item into that specific index, pushing all the following items down by one (note how “Walk the dog” got pushed from index 0 to 1).

Since you have the memory of a goldfish, you forget what you have to do, so you look at the list to get the first item:

String firstItem = todo.get(0);

The get() method returns the item at the index you specify. However, just like arrays, if your index is greater than or equal to the size of the array, you will get an exception.

Now you wonder how many items you have on your todo list. You look at it again.

int todoSize = todo.size();

The size() method simply returns the number of items currently in the ArrayList. Since the number of items can fluctuate, so can the return value over the course of a program.

You decide to start procrastinating and end up in a YouTube spiral starting from cat video compilations and ending with Beethoven’s 5th Symphony played by toothbrushes. Fully satisfied with your lack of productivity, you decide to cross it off your list.

todo.remove(0);

Now what was once your first item (on index 0) has disappeared, and all the items below it have been pushed up to fill that space. This method also returns the item it removed for convenience.

Finally, you decide that your dog doesn’t need to be walked and change the item to “Procrastinate” once again so you can continue watching cat videos.

todo.set(0, “Procrastinate”);

The set() method turns the item at the specified index (in this case, 0) into the item you want it to become (in this case, “Procrastinate”). This is sometimes useful when you only want to perform a calculation on an element without changing the order around.

To finish off this story, you end up watching YouTube for eight hours straight, resulting in a very tired you and a sad dog. The end.

Other Methods

  • toString(): prints the contents of the ArrayList in order.

  • toArray(): returns an Object array containing the items in the ArrayList. Useful for switching from ArrayList to array.
