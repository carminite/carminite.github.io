Basic OOP

Attributes and Behaviours

Object-Orientated Programming (OOP) is the concept of using classes and objects to “simulate” interactions. This provides much more flexibility than procedural (or method-calling) programming.

Each object in a program has a class which defines the properties of that object - a blueprint, if you will. When constructing an object, the program will read its corresponding class and copy its field variables (attributes) and methods (behaviours). A class also has a constructor, which basically sets up the object so that it can interact properly.

Implementation

Let’s create a class for a person:

public class Person {

}

Currently, there are no attributes or behaviours in this class, so if we try to construct this Person:

Person p = new Person();

You cannot interact with it. That’s not very useful, so let’s give this guy a name.

Attributes

public class Person {
    private String name;
}

This name is now an attribute of every Person that will be created using this class, but there’s no way we can access it without using p.name (the reason will be stated later). There are two changes we can make:

1. Constructor

public class Person {
    private String name;
    public Person(String n) {
        name = n;
    }
}

Now we can assign a name to this Person at construction.

Person p = new Person(“Joe”);

We assigned the String “Joe” to our Person’s name attribute.

Note that even when we didn’t have a constructor in our class, the Person was created without assigning attributes. This is what we call the “default constructor”. However, this default constructor will not assign any values to the object's attributes.

2. Get and Set Methods

public class Person {
    private String name;
    public Person(String n) {
        name = n;
    }
    public String getName() {
        return name;
    }
    public void setName(String n) {
        name = n;
    }
}

Now we added two methods into the class: one to get the name of the Person, and the other to set the name to another value.

System.out.println(p.getName());

p.setName("Bob");

Behaviours

Let’s give Joe here the ability to greet someone now.

public class Person {
    private String name;
    public Person(String n) {
        name = n;
    }
    public String getName() {
        return name;
    }
    public void setName(String n) {
        name = n;
    }
    public void greet() {
        System.out.println(“Hi”);
    }
}

Now we execute with:

p.greet();

Behaviours are any methods in the class. They usually interact with their attributes to process and change their states.

Customs

It’s usually custom to always make attributes private. This means that you cannot access the attributes like a field variable (Class.attribute). This makes sure that attributes are not improperly tampered with. Instead, use public get() and set() methods to access attributes.

It’s like how you would never be able to know someone’s name by looking at them, so you instead ask them for their name.
